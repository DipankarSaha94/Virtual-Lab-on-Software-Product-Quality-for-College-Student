Cyclomatic complexity is one of the metric to compute the complexity of a code. More the complexity, difficult the code to maintain. Once we get know how to compute Cyclomatic complexity, we can analyze the code and try to write less complex code which will help us to build a good quality prouct.

Cyclomatic complexity is computed using the Control Flow Graph of the program. The nodes in the graph indicate the smallest group of commands of a program, and a directed edge in it connects the two nodes i.e. if second command might immediately follow the first command.Mathematically, for a structured program, the directed graph inside control flow is the edge joining two basic blocks of the program as control may pass from first to second.
So, cyclomatic complexity M would be defined as, M = E - N + 2P
where, E = the number of edges in the control flow graph, N = the number of nodes in the control flow graph, P = the number of connected components
